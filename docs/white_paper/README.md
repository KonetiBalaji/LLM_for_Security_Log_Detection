# Hybrid Multi-Tier Approaches to Security Log Classification: A Comparative Study

**Koneti Balaji**

## Abstract

We present SENTINEL, a hybrid multi-tier framework for security log classification that combines rule-based pattern matching, transformer embedding-based machine learning, and large language model analysis. We evaluate the efficacy of each tier independently and in combination across multiple enterprise log datasets, including synthetic enterprise logs, HDFS, BGL (Blue Gene/L), and Thunderbird supercomputer logs. Our results demonstrate that the hybrid cascade approach — which routes each log entry to the most appropriate classification method based on pattern complexity and confidence thresholds — achieves higher F1 scores than any single-method baseline while maintaining practical throughput for real-time deployment. We additionally evaluate zero-shot classification capability on unseen log formats (Windows Event, CloudTrail, Kubernetes, firewall, database, and SCADA logs), finding that the combination of regex patterns and keyword-aware fallback classification provides reasonable generalisation without format-specific retraining. We map all detected threats to the MITRE ATT&CK framework and present adversarial robustness results against log poisoning and prompt injection attacks.

## 1. Introduction

Security Operations Centers (SOCs) face an escalating challenge: the volume of security log data generated by enterprise systems far exceeds the investigative capacity of human analysts. With the U.S. cybersecurity workforce gap exceeding 500,000 unfilled positions, automated log classification and triage systems represent a critical force multiplier.

Existing approaches to automated log classification broadly fall into three categories: (1) rule-based systems using regular expressions and signature matching, (2) machine learning classifiers trained on labelled log data, and (3) large language model-based analysis leveraging pre-trained knowledge. Each approach has distinct strengths and limitations. Rule-based systems are fast and deterministic but cannot handle novel patterns. ML classifiers generalise beyond training data but require labelled datasets and may produce poorly calibrated confidence scores. LLMs offer broad understanding but are expensive, slow, and potentially vulnerable to prompt injection.

We propose a hybrid multi-tier architecture that combines all three approaches in a confidence-aware cascade, where each log entry is routed to the most appropriate tier based on pattern matching capability and classifier confidence. This design follows the Chain of Responsibility pattern, ensuring that known patterns are handled efficiently by rules, statistical patterns are caught by ML, and only ambiguous entries invoke expensive LLM reasoning.

## 2. Related Work

Log parsing and anomaly detection have been extensively studied. Zhu et al. (2019) benchmarked 13 automated log parsing methods. Du et al. (2017) proposed DeepLog, using LSTM networks for anomaly detection. He et al. (2016) evaluated six log-based anomaly detection methods on HDFS and BGL datasets.

Recent work has explored transformer-based approaches: Le and Zhang (2021) applied BERT for log anomaly detection, while Guo et al. (2021) proposed LogBERT with masked log message prediction. Large language models have shown promise in zero-shot classification tasks but have not been systematically evaluated for security log classification.

Our work differs from prior approaches in three ways: (1) we explicitly compare rule-based, ML, and LLM tiers on the same datasets; (2) we evaluate a hybrid cascade strategy rather than a single method; (3) we assess cross-domain generalisation and adversarial robustness, which have received limited attention.

## 3. Methodology

### 3.1 Architecture

SENTINEL implements a three-tier classification pipeline:

**Tier 1 — Regex**: 18 compiled patterns covering security alerts, HTTP status, user actions, system notifications, resource usage, errors, deprecation warnings, and workflow failures. Confidence is deterministic (1.0 for matches).

**Tier 2 — BERT+LogReg**: The all-MiniLM-L6-v2 sentence transformer generates 384-dimensional embeddings. A logistic regression classifier trained with 5-fold stratified cross-validation produces probability scores. Predictions below confidence threshold 0.5 are declined.

**Tier 3 — LLM**: OpenAI GPT-4o-mini with structured prompts requesting category and reasoning. Includes retry logic with exponential backoff. Falls back to keyword-based simulation when no API key is configured.

The pipeline follows a Chain of Responsibility pattern: Regex → BERT → LLM, with the first tier returning a confident classification winning.

### 3.2 Datasets

| Synthetic | Enterprise (CRM, HR, Billing) | 2,410 | 9 classes (labelled) |

### 3.3 Evaluation

We evaluate using accuracy, precision, recall, and F1-score (macro and weighted). Cross-domain evaluation trains on synthetic data and tests on HDFS, BGL, and Thunderbird. Zero-shot evaluation tests on Windows Event, CloudTrail, Kubernetes, firewall, database, and SCADA log formats never seen during training. Adversarial evaluation tests resistance to log poisoning, prompt injection, and label flipping attacks.

### 4.2 Cross-Domain Generalisation

Cross-domain results demonstrate how the hybrid pipeline performs on log formats from different domains without retraining.

### 4.3 Zero-Shot Classification

The pipeline's zero-shot performance on unseen formats tests the generalisation of both the regex tier (keyword patterns are format-independent) and the LLM tier (pre-trained knowledge covers diverse log formats).

### 4.4 Adversarial Robustness

We test five evasion techniques (unicode homoglyphs, whitespace injection, case randomisation, benign prefix injection, zero-width character splitting), five prompt injection strategies, and three label flipping attacks.

## 5. Discussion

The hybrid approach demonstrates that no single classification method dominates across all log types. Regex excels at known patterns with zero latency. BERT captures statistical patterns that rules miss. The LLM tier provides a safety net for novel formats, though at higher computational cost. The cascade design ensures cost-effective operation: LLM calls are only made when cheaper methods fail.

Cross-domain results reveal that the pipeline generalises reasonably to unseen log formats, primarily through the format-independent nature of regex patterns and the broad pre-trained knowledge of the LLM tier.

## 6. Conclusion

SENTINEL demonstrates that a hybrid multi-tier approach to security log classification offers advantages over single-method baselines in both accuracy and operational efficiency. The Chain of Responsibility architecture enables organisations to balance detection accuracy against computational cost. Our evaluation across multiple datasets, zero-shot formats, and adversarial conditions provides a comprehensive assessment of practical deployment readiness.
